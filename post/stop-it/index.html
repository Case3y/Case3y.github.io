放弃折腾

2022-09-01
生活

文章目录

记录一下今天在 Windows 上折腾 VIM 编辑器的一天吧。两年前就听说了 VIM 编辑器，正好那时我还常常陷入强迫思维，我觉得有点强迫症的人都会有一点完美主义吧，至少我是这样，在一些细节上足够挑剔。

先说一下我为什么会选择去折腾 VIM 吧。刚上大学那时的我还不会陷入强迫思维的怪圈，可能有时在一些事情考虑的比较多。一开始我就感觉 VSCode 这个编辑器很不错，不仅漂亮功能也强大。但是后来我发现 VIM 之后，发现这个编辑器挺有意思，就抱着试一试的态度去折腾了一下。并且后来一段时间后的我更加挑剔，总感觉 VSCode 速度启动慢，后来更严重的时候我会觉得在 VSCode 上写代码不“优雅”，因为 VSCode 基于 chromium 内核，这让我觉得我就是在一个浏览器上写代码，这不是严格意义上的代码编辑器。

刚刚翻相册回忆起来折腾最完美的一次是在 2021 年 4 月份的时候，那时候大三上学期刚开学没多久，也就两门专业课，同时开始准备考研。所以那时候感觉时间分配还是挺充裕的，回到宿舍没事就折腾 VIM。还记得室友看到我在配置 VIM 还问我：“你怎么开始用 VIM 了？” 我回答道：“就是很想用，想配置配置” 他接着说：“这些东西我准备还是考上研之后搞吧。” 当时听到这句话后我也想了想，我现在是在干嘛，是不是在浪费时间呢？后来注意力还是在配置 VIM 上，我也就继续下去了。

深入了解之后才发现 VIM 编辑器的好多功能也是需要插件来实现，而且这些插件都读懂每一个插件的配置内容，然后自己动手一点点编写配置文件。头疼的事情来了，在遇到一些问题我总喜欢刨根问底。有些插件开发者也没有完全解释清楚所有的配置项，虽然它是起作用的，但是我的内心是不稳定的，我会感到焦虑，总担心某一天会出现很麻烦的问题。虽然最终把 VIM 配置成了自己希望的样子，刚开始还是挺开心的。我简单录了一个视频发给了我的一个朋友,他回复我：“其实这些下载一个 VSCode 就可以了😂。” 后来我也觉得是这样啊，它只是一个我完成学习和工作的工具而已，我折腾这么久在工具上，但是没有专注于学习上。并且 VSCode 和 VIM 相比真的也不赖啊。最后我卸载了已经配置好了的 VIM。

前几天疫情被封在家里好几天，也是闲的无聊了，一开始在配置博客，配置完博客之后又闲下了来，想要折腾 VIM 的心又燃起来了。这次折腾的是 NeoVIM，NeoVIM是一个巴西程序员对 VIM 的进行大幅调整的另一个版本的 VIM。今天真的是折腾了一整天，中间还简单看了一下Lua语言的语法，接着配置主题，配置LSP (language Server Protocol)和代码提示。虽然最后也配置好了，但是代码提示每一个语言都要写配置语言，结果就是我又卸载了 NeoVIM，VSCode 真香😜。

虽然折腾这么久最后还是卸载了，但是我感觉我也是有很多收获的，不只是说懂得了如何配置 VIM，在折腾的过程中一直在找教程，找文档来解决问题，自己的自学能力也得到了提高。但是仔细想想是有一点得不偿失的，自己本应该花更多时间专注于自己应该学习的知识上，专注于提升自己。这篇文章记录了自己折腾编辑器的历程，其实自己之前折腾的不只是编辑器，之前我安装 Linux 系统也是类似配置 VIM 时的心态。但是安装好 Linux 之后我又开始担心会不会不稳定，会不会出问题，最后结果就是安装回了 Windows 系统。

我觉得我这样来回折腾挺浪费时间的，也挺消耗自己的，像是精神内耗。应该把时间用到真正重要的事情，并且要合理安排好自己的时间，这几天的我时间安排的就不是很合理，几乎一直在电脑屏幕前面坐着，整日盯着文档，写着文件。总之，我选择了放弃折腾，工具只是工具，选择合适的工具完成自己本应该完成的工作也是最重要的！
